Main view

This is the central GDV hub: list the data you have uploaded, create new projects, add more data (tracks) and start working/visualizing them.

Circles

Circles allow you to setup collaborations: create a circle and add people to it, then share projects with them. To add a circle member enter his/her email via the edit link.

Sharing
You can share a GDV project:
  • Via circles: share within a group of collaborators. the project will appear in the shared project section on the main interface of all users within the circle. You can give three different permissions to a circle:
    • Read: User can view the data in the project but not download or add new data to it
    • Download: User can view and download the project data.
    • Upload: User can edit the project and add or delete tracks.
  • Via links: provide public or download access links.
    • The public link gives access to view a project. This link allows a user to see all tracks in the project but not to download or modify the data.
    • The download link enables to copy a project and gives permanent access to the project data.
Jobs

You can launch computations from within GDV: in the project view, choose an operation from the Data panel and apply it to selected tracks, then fetch the results when the operation is finished.

Assemblies

Additional assemblies can can be provided on demand by email.

API

There a simple programming interface to GDV, based on HTTP requests to create projects, add tracks, and prepare view without the web interface. For more information read our python library documentation. There is a python script that you could use for this purpose. It's part of the bbcf utilities.

Quick start:

  • wget https://raw.github.com/bbcf/bbcfutils/master/Python/pygdv.py (fetch the file from github)
  • easy_install clize (install a third-party library - perhaps you will need a sudo)
  • python pygdv --help

To use the API you need to authenticate with your personal API key that you can retrieve on your user page.
